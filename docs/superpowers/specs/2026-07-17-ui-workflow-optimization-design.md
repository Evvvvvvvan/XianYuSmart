# UI 与核心流程优化设计

## 目标

在不改动现有业务边界的前提下，统一界面基础样式、减少首屏空白和重复请求、让关键操作拥有明确的加载与结果反馈，并保证桌面端、平板端和移动端操作一致。

## 现状与根因

- 应用只加载 `theme.css`，基础重置与提示动画所在的 `main.css` 未生效，页面间距和控件表现不一致。
- 全局提示与确认框直接拼接动态文本，既不安全，也缺少键盘、无障碍和重复提示处理。
- 运营中心概览分别拉取十类资源和任务列表，首屏产生 11 次请求；执行按钮也没有防重复状态。
- 布局组件挂载后才判断屏幕类型，首次渲染会短暂空白；内容区和页面自身重复留白。
- 少量页面仍调用浏览器原生弹窗，体验与系统组件不一致。

## 方案

1. 由 `main.css` 统一加载基础重置与商业主题，仅保留必要阴影和过渡，不使用大面积装饰。
2. 布局在首次渲染前确定设备类型，页面标题直接读取路由元数据，主内容区只负责滚动。
3. 提示与确认框改为安全 DOM 构建，支持换行、重复消息合并、Esc/Enter 和焦点管理。
4. 新增运营概览聚合接口，通过两次聚合查询返回资源数、任务数和失败数，前端只请求一次。
5. 运营操作增加逐项忙碌状态、危险执行确认、失败任务结果反馈和保存防重复。
6. 原生 `alert`、`confirm` 替换为现有统一反馈组件。

## 边界

- 不改变已有数据库结构和业务调度规则。
- 不引入新的前端依赖和视觉组件库。
- 不调整无关页面业务逻辑，不迭代 Release。
